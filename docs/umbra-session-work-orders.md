# Umbra Session Work Orders

## 목적

이 문서는 Umbra를 컨텍스트 프리한 세션 단위로 개발할 수 있도록 쪼갠 작업 지시서 모음이다. 각 세션은 새 대화에서 단독으로 실행할 수 있도록 설계되었고, 이전 대화 맥락 없이도 저장소 상태를 먼저 조사한 뒤 끝까지 구현할 수 있게 작성되어 있다.

## 사용법

새 세션에서 아래처럼 말하면 된다.

```text
C:\Users\CrowKing63\Developments\Umbra\docs\umbra-session-work-orders.md를 읽고 UMB-04만 수행해줘.
```

또는 각 섹션의 `복붙용 프롬프트` 블록만 그대로 보내도 된다.

## 공통 전제

- 제품명은 Umbra다.
- Umbra는 Windows에서 백그라운드로 실행되는 local-first Markdown writing server다.
- 사용자는 브라우저로 `localhost` 또는 로컬 IP를 통해 접속한다.
- 문서는 사용자가 지정한 로컬 폴더 아래의 `.md` 파일 중심으로 저장된다.
- 권장 스택은 `Electron + Fastify + React + TypeScript + CodeMirror 6 + SQLite`다.
- 문서 내용은 가능한 한 사용자 문서 루트에 두고, 앱 메타데이터와 히스토리 스냅샷은 앱 데이터 디렉터리에 둔다.
- LAN 접근은 기본 비활성화가 원칙이다.
- UI는 touch-friendly해야 하며 Apple Vision Pro Safari도 고려한다.
- 각 세션은 저장소를 먼저 조사하고, 필요한 최소 범위만 구현하며, 가능한 경우 검증까지 수행해야 한다.

## UMB-02. Electron 데스크톱 셸 구성

### 완료 조건

- Electron 메인 프로세스가 있다.
- 시스템 트레이 메뉴가 있다.
- 앱 시작 시 로컬 URL을 열 수 있다.
- 최소한의 auto-start 설정 훅 또는 placeholder가 있다.

### 검증 기준

- 데스크톱 앱 실행 시 트레이 또는 메인 창이 뜬다.
- 트레이에서 앱 열기/종료가 동작한다.

### 복붙용 프롬프트

```text
너는 C:\Users\CrowKing63\Developments\Umbra 저장소에서 Umbra의 UMB-02 작업만 수행한다.

제품 배경:
- Umbra는 Windows에서 백그라운드로 실행되는 local-first Markdown writing server다.
- Electron은 OS 통합, 트레이, 자동 시작, 로컬 서버 라이프사이클 제어를 맡는다.
- 웹 UI는 localhost 또는 나중에 LAN 주소로 열린다.

이번 세션 목표:
- Electron 메인 프로세스를 구성한다.

완료 조건:

작업 후에는 변경 내용, 검증 결과, 남은 리스크를 짧게 보고해라.
```

- 설정을 변경하고 재시작해도 유지된다.

### 복붙용 프롬프트

```text
너는 C:\Users\CrowKing63\Developments\Umbra 저장소에서 Umbra의 UMB-03 작업만 수행한다.
```

```text
- 현재 로컬 IP와 접속 URL을 계산해 표시하는 로직을 만든다.
- 웹 설정 화면 또는 Electron 트레이에서 localhost/LAN URL을 확인하고 복사할 수 있게 한다.
- 가능하면 LAN enable 시 password enable을 권장하거나 경고한다.

구현 원칙:
- 보안 기본값은 항상 보수적으로 유지한다.
- 플랫폼별 네트워크 주소 계산이 필요하면 현재 환경에서 합리적인 방법으로 구현한다.

완료 조건:
- 사용자가 LAN 공개 여부를 명확히 제어할 수 있다.
- 로컬 장치에서 접속할 URL을 쉽게 확인할 수 있다.

작업 후에는 변경 내용, 검증 결과, 남은 리스크를 짧게 보고해라.
```
